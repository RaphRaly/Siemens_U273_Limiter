2. Noeuds et inconnues MNA
--------------------------

Noeuds :

    0    = masse locale
    VS   = source equivalente apres U1, avant R5
    NX   = noeud entre R5 et C1
    NA   = noeud audio apres C1, marque 25 mV~
    NB   = noeud de commande couple a NA par C2
    NL   = noeud interne gauche du pont
    NR   = noeud interne droit du pont
    N14  = bas gauche du pont
    N15  = bas droit du pont
    NM   = noeud d'equilibrage R8/R10
    VCMD = commande venant de B11

Vecteur minimal si VS et VCMD sont traites comme sources imposees :

    x = [
        vNX,
        vNA,
        vNB,
        vNL,
        vNR,
        vN14,
        vN15,
        vNM
    ]^T

Si VS ou VCMD sont modelises comme sources de tension ideales dans la MNA, il faut
ajouter leurs courants inconnus :

    x = [ tensions nodales ; iVS ; iVCMD ]^T


3. Regimes a ne pas melanger
----------------------------

DC :

    C1, C2, C3, C4, C5, C6, C7 ouverts.

But : trouver le point de repos des diodes sous une commande VCMD.

Petit signal AC :

    les diodes deviennent gd autour du point DC ;
    les condensateurs deviennent sC ;
    on calcule l'impedance equivalente du pont et l'attenuation locale.

Temporel DSP :

    les diodes sont linearisees a chaque sample ;
    les condensateurs deviennent des compagnons gc/iceq ;
    Newton-Raphson resout le systeme implicite.


4. Stamps lineaires
-------------------

Resistance R entre a et b :

    g = 1/R
    i_ab = g*(va - vb)

Stamp :

    G[a,a] += g
    G[b,b] += g
    G[a,b] -= g
    G[b,a] -= g

Si b = masse :

    G[a,a] += g


5. Diodes : linearisation Newton
--------------------------------

Modele Shockley generique :

    id(vd) = Is * (exp(vd/(n*VT)) - 1)
    vd = va - vk

Au point Newton vd0 :

    id0 = Is * (exp(vd0/(n*VT)) - 1)
    gd  = Is/(n*VT) * exp(vd0/(n*VT))
    ieq = id0 - gd*vd0

Modele local :

    id ~= gd*vd + ieq

Stamp pour courant positif de a vers k :

    G recoit gd entre a et k
    I[a] -= ieq
    I[k] += ieq

Important :

    Is et n doivent rester parametriques.

Proposition :

    OA154Q : Is_OA, n_OA
    SSD55  : Is_SS, n_SS

Ces valeurs ne doivent pas etre inventees depuis le guide generique. Elles doivent venir
d'une mesure, d'une datasheet, ou d'une calibration sur appareil.

6. Condensateurs : compagnon numerique
--------------------------------------

Le fichier analog_modeling_guide.md recommande une formulation mix/trapezoidale pour
l'audio. On retient :

    m = 1/2  pour trapezoidal/Tustin
    m = 1    pour Backward Euler de debug

Pour un condensateur C entre a et b, orientation a -> b :

    vC[n] = va[n] - vb[n]
    h     = 1 / sampleRate
    minv  = 1 / m
    gc    = C / (h*m)

Modele Norton :

    iC[n] = gc*vC[n] - iceq[n]

Mise a jour apres convergence Newton :

    iceq[n+1] = iceq[n] + minv*(gc*vC[n] - iceq[n])

Avec m = 1/2 :

    gc = 2*C*sampleRate

Backward Euler est plus stable mais plus amorti. Il peut servir pour verifier le solveur,
pas comme choix final automatique.


7. KCL principales
------------------

Les orientations exactes des diodes et des potentiometres doivent etre confirmees par une
netlist finale. Les equations ci-dessous donnent la structure MNA correcte.

NX :

    0 = g5*(vNX - vVS)
      + gc1*(vNX - vNA) - iceq_C1

NA :

    0 = gc1*(vNA - vNX) + iceq_C1
      + gc2*(vNA - vNB) - iceq_C2
      + Yin_amp*vNA

NB :

    0 = gc2*(vNB - vNA) + iceq_C2
      + id3(NB, NL)
      + id2(NB, NR)
      + I_Rtrim_NB

NL :

    0 = id4(NL, N14)
      - id3(NB, NL)
      + I_Rtrim_NL

NR :

    0 = id1(NR, N15)
      - id2(NB, NR)
      + I_Rtrim_NR

N14 :

    0 = -id4(NL, N14)
      + g6*vN14
      + gc3*vN14 - iceq_C3
      + gc4*vN14 - iceq_C4
      + gc5*(vN14 - vN15) - iceq_C5
      + I_commande_14

N15 :

    0 = -id1(NR, N15)
      + g11*vN15
      + gc7*vN15 - iceq_C7
      + gc6*vN15 - iceq_C6
      + gc5*(vN15 - vN14) + iceq_C5
      + I_commande_15

NM :

    0 = somme_j (vNM - vj)/Rj + sources eventuelles de commande

NM depend de la connexion exacte de R8/R10 et des curseurs. Il ne faut pas figer cette
equation avant d'avoir transforme R7/R8 en netlist precise.


8. Potentiometres et ajustables
-------------------------------

Un potentiometre ne doit pas etre remplace par une seule resistance fixe.

Pour R7 = 100 Ohm :

    alpha7 in [0,1]
    R7a = max(alpha7*100, Rmin)
    R7b = max((1-alpha7)*100, Rmin)

Pour R8 = 250 kOhm :

    alpha8 in [0,1]
    R8a = max(alpha8*250k, Rmin)
    R8b = max((1-alpha8)*250k, Rmin)

Rmin evite une conductance infinie quand le curseur est en butee.

C4 et C6 doivent rester parametriques :

    C4 = c4_abgl
    C6 = c6_abgl


9. Assemblage MNA par sample
----------------------------

Pour chaque sample :

    1. utiliser la solution du sample precedent comme point initial ;
    2. remettre G et I a zero ;
    3. stamp les resistances ;
    4. stamp les condensateurs avec gc/iceq ;
    5. calculer vd0 pour D1-D4 ;
    6. calculer gd/ieq pour D1-D4 ;
    7. stamp les diodes linearisees ;
    8. stamp les sources VS et VCMD ;
    9. resoudre G*x = I ;
    10. repeter jusqu'a convergence ;
    11. mettre a jour les iceq des condensateurs.

Critere pratique :

    max(abs(x_new - x_old)) < 1e-7 V

Limite :

    20 a 50 iterations maximum

En pratique audio, avec le sample precedent comme guess, on doit viser plutot :

    2 a 6 iterations


10. Pseudo-code
---------------

    init:
        charger R, C, Is, n, sampleRate
        m = 0.5
        calculer gc pour chaque condensateur
        iceq_C1..iceq_C7 = 0
        x_prev = 0

    tick(VS, VCMD):
        x = x_prev

        for iter in 1..Nmax:
            G = 0
            I = 0

            stamp_resistances(G, I)
            stamp_capacitors_mix(G, I, iceq, m)
            stamp_sources(G, I, VS, VCMD)

            for diode in D1..D4:
                vd0 = x[anode] - x[cathode]
                vd0 = clamp(vd0, -1.0, +1.0)
                id0 = Is*(exp(vd0/(n*VT)) - 1)
                gd  = Is/(n*VT)*exp(vd0/(n*VT))
                ieq = id0 - gd*vd0
                stamp_diode(G, I, anode, cathode, gd, ieq)

            x_new = solve(G, I)

            if max_abs(x_new - x) < tolerance:
                x = x_new
                break

            x = x + lambda*(x_new - x)

        update_iceq(x)
        x_prev = x
        return vNA, autres noeuds utiles

Le facteur lambda est un damping Newton :

    0.1 <= lambda <= 1

Il aide quand les exponentielles rendent l'iteration trop agressive.
