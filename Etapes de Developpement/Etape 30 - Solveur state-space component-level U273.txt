Etape 30 - Solveur state-space component-level U273
===================================================

Date: 2026-05-04
Objet: ajouter le premier moteur numerique state-space / DAE component-level
       pour le modele analogique final du Siemens U273, sans WDF et sans
       modelisation transformateur magnetique dans ce sprint.


1. Decision scientifique
------------------------

Le moteur final doit etre formule comme un systeme dynamique implicite:

    F(z, zdot, u, p) = 0

avec une base MNA/DAE:

    Q(z) * dz/dt + G(z,p) + i_nl(z,p) = B*u

La discretisation initiale privilegie backward Euler pour sa robustesse:

    R(z_k) = Q(z_k) * (z_k - z_(k-1)) / h + G(z_k) + i_nl(z_k) - B*u_k = 0

Chaque pas est resolu par Newton-Raphson amorti:

    J(z_n) * delta = -R(z_n)
    z_(n+1) = z_n + alpha * delta

Le trapeze est implemente pour les condensateurs, mais backward Euler reste le
choix par defaut tant que la fermeture non-lineaire U273 n'est pas calibree.


2. Implementation C++
---------------------

Ajout dans:

    u273-plugin/modules/u273_reference/include/u273/reference/state_space
    u273-plugin/modules/u273_reference/src/state_space

Briques creees:

Le solveur reste offline dans u273_reference. Le chemin realtime
u273_dsp/u273_plugin n'est pas remplace a cette etape.


3. Composants couverts
----------------------

Stamps disponibles:

    resistance;
    condensateur backward Euler et trapezoidal companion;
    source courant;
    source tension ideale;
    diode Shockley avec conductance analytique;
    NPN Ebers-Moll simplifie avec jacobien analytique;
    metadata port transformateur ideal.

Les transformateurs U1/U2 sont volontairement exclus du modele magnetique:

    port ideal fige;
    pas de saturation;
    pas d'hysteresis;
    pas de fuite;
    pas d'inductance magnetisante;
    pas de technique personnelle transformateur dans ce sprint.
